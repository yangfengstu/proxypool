# ProxyPool 核心机制详解

## 🎯 健康评分算法

### 评分公式

```go
healthScore = 成功率×60% - 超时率×20% + 延迟评分×20%

// 连续失败惩罚
if 连续失败次数 >= 3 {
    healthScore = healthScore × 0.5
}

// 最终范围: 0.0 ~ 1.0
```

### 详细计算过程

#### 1. 成功率（60%权重）
```go
successRate = 成功次数 / 总使用次数

示例：
- 使用10次，成功8次 → successRate = 0.8
- 贡献分数: 0.8 × 0.6 = 0.48
```

#### 2. 超时率（-20%权重，负面指标）
```go
timeoutRate = 超时次数 / 总使用次数

示例：
- 使用10次，超时2次 → timeoutRate = 0.2
- 扣除分数: 0.2 × 0.2 = 0.04（从总分中减去）
```

#### 3. 延迟评分（20%权重）
```go
avgLatency = 总延迟时间 / 总使用次数

延迟评分规则：
- avgLatency <= 1秒  → latencyScore = 1.0  (优秀)
- 1秒 < avgLatency <= 3秒 → latencyScore = 0.8  (良好)
- 3秒 < avgLatency <= 5秒 → latencyScore = 0.6  (一般)
- avgLatency > 5秒      → latencyScore = 0.3  (较差)

贡献分数: latencyScore × 0.2
```

#### 4. 连续失败惩罚
```go
if 连续失败次数 >= 3 {
    finalScore = healthScore × 0.5  // 减半
}
```

### 完整示例

#### 示例1：健康代理
```
使用情况：
- 总使用: 20次
- 成功: 18次
- 失败: 2次
- 超时: 1次
- 总延迟: 10秒
- 连续失败: 0次

计算：
成功率 = 18/20 = 0.9           → 0.9 × 0.6 = 0.54
超时率 = 1/20 = 0.05           → 0.05 × 0.2 = 0.01 (扣除)
平均延迟 = 10/20 = 0.5秒 < 1秒  → 1.0 × 0.2 = 0.20
连续失败 = 0                   → 无惩罚

健康评分 = 0.54 - 0.01 + 0.20 = 0.73 ✓ (健康)
```

#### 示例2：一般代理
```
使用情况：
- 总使用: 20次
- 成功: 15次
- 失败: 5次
- 超时: 3次
- 总延迟: 50秒
- 连续失败: 2次

计算：
成功率 = 15/20 = 0.75          → 0.75 × 0.6 = 0.45
超时率 = 3/20 = 0.15           → 0.15 × 0.2 = 0.03 (扣除)
平均延迟 = 50/20 = 2.5秒       → 0.8 × 0.2 = 0.16
连续失败 = 2                   → 无惩罚 (< 3)

健康评分 = 0.45 - 0.03 + 0.16 = 0.58 ✓ (一般)
```

#### 示例3：不健康代理
```
使用情况：
- 总使用: 20次
- 成功: 8次
- 失败: 12次
- 超时: 8次
- 总延迟: 100秒
- 连续失败: 4次

计算：
成功率 = 8/20 = 0.4            → 0.4 × 0.6 = 0.24
超时率 = 8/20 = 0.4            → 0.4 × 0.2 = 0.08 (扣除)
平均延迟 = 100/20 = 5秒        → 0.3 × 0.2 = 0.06
连续失败 = 4 >= 3              → 减半惩罚

健康评分 = (0.24 - 0.08 + 0.06) × 0.5 = 0.11 ✗ (不健康，会被剔除)
```

### 剔除阈值

```go
// 代码位置: background.go, pruneUnhealthyProxies()

剔除条件（满足任一即剔除）：
1. healthScore < 0.3           // 健康评分过低（默认）
2. 连续失败 >= 5次              // 连续失败过多（默认）
3. 失败率 > 80% (使用>10次)    // 总体失败率过高（默认）
```

## 🔄 池大小维护机制

### 水位线策略

```go
// 配置（types.go, Config结构）
TargetSize:    300,    // 目标池大小
LowWatermark:  0.7,    // 70%低水位 (210个)
HighWatermark: 0.9,    // 90%高水位 (270个)
```

### 监控频率

```go
// 代码位置: types.go, Config.MonitorInterval
MonitorInterval: 1 * time.Minute  // 默认每1分钟检查一次
```

### 检查逻辑

```go
// 代码位置: background.go, checkAndRefresh()

每1分钟执行：
1. 统计可用代理数（未过期 + 连续失败<5）
2. 统计即将过期代理数（未来15分钟内过期）
3. 根据情况决定补充策略

补充策略：
┌─────────────────────────────────────────┐
│ 场景1: 可用数 < 210 (70%)               │
│ 原因: URGENT - 低于低水位                │
│ 动作: 立即补充到300个                    │
│ 拉取: 300 - 当前可用数                   │
├─────────────────────────────────────────┤
│ 场景2: 即将过期 > 20个                   │
│ 原因: PROACTIVE - 预防性补充             │
│ 动作: 提前补充                          │
│ 拉取: min(即将过期数, 40)                │
├─────────────────────────────────────────┤
│ 场景3: 210 <= 可用数 < 270 (90%)        │
│ 原因: MAINTAIN - 维持高水位              │
│ 动作: 补充到270个                        │
│ 拉取: 270 - 当前可用数                   │
├─────────────────────────────────────────┤
│ 场景4: 可用数 >= 270                     │
│ 原因: 健康状态                           │
│ 动作: 不补充                             │
│ 拉取: 0                                  │
└─────────────────────────────────────────┘
```

## 🔧 不健康代理剔除机制

### 剔除频率

```go
// 代码位置: types.go, Config.PruneInterval
PruneInterval: 2 * time.Minute  // 默认每2分钟检查一次
```

### 剔除流程

```go
// 代码位置: background.go, pruneTask()

每2分钟执行：
1. 遍历所有代理
2. 更新健康评分（updateHealthScore）
3. 检查剔除条件
4. 移除不健康代理
5. 触发水位线检查（可能补充）

剔除后立即检查：
if 剔除了代理 {
    立即调用 checkAndRefresh()
    // 确保池大小维持在目标范围
}
```

## 📊 配置建议

### 推荐配置（生产环境）

```go
Config{
    // 池大小
    TargetSize:    300,
    LowWatermark:  0.7,   // 70% = 210个
    HighWatermark: 0.9,   // 90% = 270个
    
    // 监控频率
    MonitorInterval: 1 * time.Minute,  // 每1分钟检查水位
    PruneInterval:   2 * time.Minute,  // 每2分钟剔除不健康
    
    // 健康阈值
    MinHealthScore:      0.3,  // 低于0.3剔除
    MaxConsecutiveFails: 5,    // 连续失败5次剔除
    MaxFailRate:         0.8,  // 失败率80%剔除
    
    // 预防性刷新
    RefreshWindow: 15 * time.Minute,  // 提前15分钟补充
}
```

### 根据规模调整

#### 小规模（<100个代理）
```go
TargetSize:      50,
MonitorInterval: 30 * time.Second,  // 更频繁检查
PruneInterval:   1 * time.Minute,
```

#### 中规模（100-300个代理）
```go
TargetSize:      200,
MonitorInterval: 1 * time.Minute,   // 默认
PruneInterval:   2 * time.Minute,
```

#### 大规模（>300个代理）
```go
TargetSize:      500,
MonitorInterval: 2 * time.Minute,   // 减少检查频率
PruneInterval:   5 * time.Minute,
```

## 🎯 总结

### 核心机制

1. **健康评分**
   - 每次使用后更新统计
   - 每2分钟重新计算评分
   - 评分<0.3自动剔除

2. **池大小维护**
   - 每1分钟检查一次
   - 水位线自动补充（70%-90%）
   - 提前15分钟预防性刷新

3. **自动优化**
   - 连续失败立即惩罚（评分减半）
   - 连续失败5次立即剔除
   - 失败率>80%自动剔除

### 时间周期总结

| 操作 | 频率 | 位置 |
|------|------|------|
| 水位线检查 | 每1分钟 | MonitorInterval |
| 不健康剔除 | 每2分钟 | PruneInterval |
| 健康评分更新 | 每2分钟 | pruneTask |
| 预防性刷新 | 提前15分钟 | RefreshWindow |
| 健康检查 | 每5分钟 | HealthCheckInterval (可选) |

**所有频率都可以通过Config配置调整！**
